[4/24/2021 2:59â€¯AM] tux_linux
Boot process is as follows:
- Kernel
- Initrd (with busybox static binary for recovery purposes) which waits for a power button event, and if there is, "boots" to the second partition (recovery) by chrooting and mounting appropriate filesystems. (mmcblk0p2)
- Rootfs: Alpine 3.10 rootfs, fully bootable natively, but now I just chroot into it like the diags partition. (mmcblk0p3)
- Multiple chroots for various services: one for Xorg (Alpine 3.10), one for VNC/X11 touch input (Debian 8), and one for InkBox itself which runs a 2015 buildroot build since the glibc version supports the kernel correctly (2.6.35.3)
- Updates in the isa format are actually squashfs-compressed folders (usually in XZ, where possible) and are mounted via squashFUSE on boot. An extensions system for X11 allows seamless add/delete operations for extensions, e.g., netsurf.isa is an extension (which is like a mini root with the program and its deps), and the initrd detects it on boot and merges it (unionfs-fuse) with the main X.org rootfs, also a Squashfs, but overlayfs-fuse mounted so it can be writeable.
